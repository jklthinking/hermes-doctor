# PRD Summary

Hermes Doctor 的功能目标对齐皮皮虾医生 PRD，但实现基于 Hermes 自身插件框架。

## Product Positioning

面向 Hermes Agent 用户的自诊断与安全自愈插件。它不是 AgentOps 大平台，而是一个可安装、可测试、可沉淀经验的 Hermes Skill。

## Target Users

- 不会读日志、终端、配置和依赖错误的小白用户。
- 需要快速体检和复盘记录的 Hermes 维护者。
- 需要把常见问题沉淀成药方的团队成员。

## MVP Scope

1. Hermes 插件结构体检。
2. 错误文本到药方匹配。
3. 小白可读诊断报告。
4. 修复风险分级与确认策略。
5. 修复计划生成，不直接执行。
6. 脱敏病历记录。
7. 病历搜索。
8. 飞书/Lark 消息路由。
9. 一键 smoke tests。

## Boundaries

可以做：

- 诊断 Hermes 插件/Agent/工具/配置/日志/依赖问题。
- 解释问题影响和下一步。
- 生成修复计划。
- 在确认后执行低风险修复。
- 记录可复用病历。

不能做：

- 保证零故障。
- 绕过登录、验证码、反爬、限流。
- 收集无关隐私或秘密。
- 未经确认执行破坏性操作。

## Acceptance Criteria

- 体检 5 秒内返回。
- 报告包含严重度、影响范围、证据、药方、风险和下一步。
- 药方覆盖结构、JSON、依赖、工具调用、网页数据、登录限流、飞书路由和安全修复。
- 写入、安装、授权、重启、配置修改前必须确认。
- 一键测试通过。
