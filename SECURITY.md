# Security Policy

## Reporting a Vulnerability

Please do **not** open a public GitHub issue for suspected security vulnerabilities.

Report security issues through one of the following channels:

1. Open a private security advisory in this repository when available.
2. Contact the maintainers through the official AtomCollide / 智械工坊 communication channels listed in the product matrix.

When reporting, include:

- Affected repository and version/commit.
- Reproduction steps or proof of concept.
- Expected impact and affected users/data.
- Suggested remediation if known.

## Supported Versions

Unless a repository-specific policy states otherwise, only the default branch and latest published release are actively maintained.

## Disclosure

We follow coordinated disclosure. Maintainers will triage reports, prioritize fixes by severity, and publish remediation notes when a fix is available.
